## Irodori-TTS の解説

### 概要
- **Irodori-TTS** - Flow Matching ベースの TTS（音声合成）モデル
- 絵文字で感情・スタイルを制御できる日本語対応 TTS
- 500M / 2.5B パラメータの 2 モデル
- ゼロショットボイスクローニング対応
- HuggingFace にデモページあり

### アーキテクチャ
- **テキストエンコーダー**: 事前学習済み LLM のトークン埋め込み + RoPE + SwiGLU
- **参照潜在エンコーダー**: 話者・スタイル条件付け
- **Diffusion Transformer (DiT)**: Flow Matching + Low-Rank AdaLN

### 必要スペック
- **推論（500M）**: GTX 1060 6GB 以上 / VRAM 4-6GB
- **学習（500M）**: RTX 3090 24GB 以上推奨 / バッチサイズ小さくすれば 16GB も可
- **macOS**: M1/M2/M3 対応（MPS）
- **CPU のみ**: 非常に遅い（テスト用途のみ）

## 音声合成のライセンス問題

### VOICEVOX のライセンス
- オープンソースプロジェクト
- 各キャラクターごとに利用規約が異なる
- ずんだもん、四国めたんなどは商用利用 OK
- **学習データとしての使用は明確に許可されていない場合が多い**
- 推論（ゼロショットクローニング）での使用は OK な場合が多い

### 安全な学習用音声素材
- **CoeFont** - クラウド音声合成、学習用データセット提供
- **Otoya** - フリー音声素材、ライセンス明確
- **Audiostock** - 音声素材サイト、商用利用 OK 多数

## Rust GUI フレームワーク議論

### GPUI（Zed エディタの UI フレームワーク）
- GPU アクセラレーション対応
- ハイブリッド（即時 + 保持）モード
- **Windows 未対応**（macOS/Linux のみ）

### egui
- Rust 製即時モード GUI ライブラリ
- **Windows / macOS / Linux / Web / Android 対応**
- シンプルで使いやすい
- ゲームエンジン統合可能
- Rerun（データ可視化 SDK）のスポンサー
